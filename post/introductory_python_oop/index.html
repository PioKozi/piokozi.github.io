Introduction to OOP in Python

This serves only as an introduction to basic parts of OOP in Python 3.

Defining a class

We can define a class using the class keyword. We can give this class some variables and methods.

#!/usr/bin/env python

class Foo:
    bar = "hello world"
    def print_var(self):
        print(self.bar)

f = Foo()
f.print_var()

hello world

Notice here, that print_var takes self as a parameter, however we do not have to pass it as an argument. Notice also, that when we call bar inside of any method in Foo, we must prepend it with self..

Constructor Method

We might want to pass some arguments to the class when defining an instance of it. For this, we need a constructor method. Python uses a method called __init__ to allow this.

#!/usr/bin/env python

class Foo:
    def __init__(self, val):
        self.bar = val

f = Foo("foobar!")
print(f.bar)

foobar!

Notice that here we have passed the value "foobar!" as an argument, which is then passed into the class variable bar. We can also access this variable outside of the class, as we do in f.bar.

Public and Private

Unfortunately Python has no real notion of public or private methods/variables, however by convention anything beginning with an underscore is considered to be private, however it can also just mean what is being done shouldn’t affect any of the children of a class. Raymond Hettinger has a much better explanation of this, which you should watch if you are interested.

Inheritance

A class can inherit all the methods and variables of another class, here is a basic example.

#!/usr/bin/env python

class Foo:
    def say_hello(self):
        print("hello world!")

class Bar(Foo):
    pass

b = Bar()
b.say_hello()

hello world!

The line class Bar(Foo) means that Bar is inheriting from Foo. Notice this, we haven’t defined any say_hello method for Bar, however as we have done so for Foo and Bar inherits from Foo, the method exists and works.

Polymorphism

Very generally, this means we can overwrite something we made in a superclass (parent class).

#!/usr/bin/env python

class Foo:
    def say_hello(self):
        print("hello world!")

class Bar(Foo):
    def say_hello(self):
        print("Lorem Ipsum...")

f = Foo()
f.say_hello()
b = Bar()
b.say_hello()

hello world!
Lorem Ipsum...

Notice that although Bar inherits from Foo, the method of the same name does something different.

Magic Methods

In python, there are methods which begin and end with two underscores (called a dunder); these are called “magic methods” and are not supposed to be called by the programmer. We have already seen an example of this with the __init__ method! Notice that we did not have to call __init__ to make it run, rather it ran when we defined the class. Generally speaking, they define how common interactions with the class should take place.

I don’t intend to go in depth of what magic methods exist, only that you can define your own. If you want to go more in-depth into magic methods, I recommend looking at the relevant python documentation.
